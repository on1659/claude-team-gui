# Claude 팀원 GUI — 프로젝트 개요

> CLI 스킬로 검증한 "AI 팀원 시스템" 개념을 GUI 앱으로 구현하는 프로젝트

---

## 1. 핵심 개념

### Claude 멀티에이전트란?

Claude는 단일 인스턴스 외에도 **여러 인스턴스가 독립적으로 병렬 실행**되는 구조로 운용할 수 있다.
오케스트레이터(주 Claude)가 Agent 도구로 서브에이전트를 생성하고, 결과를 수집·종합한다.

```
오케스트레이터 Claude
    ↓ Agent 도구 (동시 실행)
서브에이전트 A  서브에이전트 B  서브에이전트 C
(독립 컨텍스트) (독립 컨텍스트) (독립 컨텍스트)
    ↓               ↓               ↓
        결과 반환 → 오케스트레이터 종합
```

### 서브에이전트 수명

에이전트는 기본적으로 **일회용(프리랜서)**이다.

```
생성 → 작업 실행 → 결과 반환 → 소멸
```

`resume` 파라미터로 재소환 가능:
```
1차 실행 → agent_id: "abc123" 반환
resume: "abc123" → 이전 컨텍스트 그대로 이어서 실행
```

단, 세션이 끊기면 resume도 불가능 → 장기 지속은 프로필 파일로 해결

## 2. 회의 방식 3종

### 방식 비교

| 스킬 | 구조 | 인원 | 특징 |
|------|------|------|------|
| `/meeting-multi` | Claude 1개, 순차, 누적 | 기본 8명 | 자연스러운 토론 흐름, 편향 가능 |
| `/meeting-agent` | Claude 여러 개, 병렬 | 기본 8명 | 독립 분석, 편향 없음 |
| `/meeting-team` | Claude 여러 개, 병렬 + 프로필 주입 | 기본 8명 | 팀원 정체성 기반, 가장 일관됨 |

### 역할극 방식 (`/meeting-multi`)

```
[Claude 1개, 단일 컨텍스트]

→ "저는 기획자입니다. 이 기능은 사용자 니즈에 부합합니다."
→ "저는 개발자입니다. 기술적으로 가능합니다."  ← 기획자 말 읽고 답변
→ "저는 QA입니다. 다음 리스크가 있습니다."     ← 앞 두 명 말 읽고 답변
```

- 앞 발언이 뒤 역할에 영향 → **편향 발생 가능**
- 인원 증가 시 컨텍스트 누적 → **토큰 폭증**

### 진짜 멀티에이전트 방식 (`/meeting-agent`, `/meeting-team`)

```
[오케스트레이터]
    ↓ 동시 실행
[기획자①] [기획자②] [개발자①] [개발자②] [QA] [UI] [UX] [PD]
  주제만     주제만     주제만     주제만    ...
  독립분석   독립분석   독립분석   독립분석
    ↓ 모든 결과 수집
[오케스트레이터] → 최종 종합
```

- 서로의 발언을 **전혀 모르는 상태**에서 독립 분석
- 편향 없음 / 토큰 선형 증가

## 3. 토큰 비용

### 절대값 (8인 기준)

| 스킬 | input | output | 총합 |
|------|-------|--------|------|
| `/meeting-agent` | ~8,700 | ~2,900 | **~11,600** |
| `/meeting-team` | ~14,300 | ~3,800 | **~18,100** |
| `/meeting-multi` | ~16,300 | ~2,900 | **~19,200** |

### 인원별 효율 (역할극 vs 멀티에이전트)

| 인원 | `/meeting-multi` | `/meeting-agent` |
|------|-----------------|-----------------|
| 2명 | ~3,900 | ~3,800 |
| 4명 | ~8,100 | ~6,000 |
| 8명 | ~19,200 | ~11,600 |

> 역할극은 인원 증가 시 **제곱에 가깝게 폭증**, 멀티에이전트는 **선형 증가**.
> 8명 기준 멀티에이전트가 역할극보다 40% 저렴.

### 언제 어떤 방식을?

| 상황 | 추천 |
|------|------|
| 빠른 검토, 자연스러운 흐름 | `/meeting-multi` |
| 편향 없는 독립 의견 / 5명 이상 | `/meeting-agent` |
| 중요한 결정, 일관된 팀원 캐릭터 | `/meeting-team` |

## 4. 팀원 고용 체계

### 고용 형태

| 형태 | 기술적 실체 | 범위 | 특징 |
|------|------------|------|------|
| **정규직** | `~/.claude/` 글로벌 팀원 | 전체 컴퓨터 | 모든 프로젝트, 해고 전까지 유지 |
| **비정규직** | `.claude/` 프로젝트 팀원 | 프로젝트 한정 | 프로젝트 종료 시 계약 만료 |
| **프리랜서** | 서브에이전트 (일회성) | 세션 내 | 일 맡기면 하고 사라짐 |
| **재계약** | `resume` 파라미터 | 세션 내 | 프리랜서 재소환, 이전 컨텍스트 유지 |

부가 용어: **연봉** = 토큰 사용량 / **발령** = 글로벌 ↔ 프로젝트 이동 / **퇴사** = 프로필 삭제

### 현재 팀원 구성 (기본 8명)

| 이름 | 직군 | 판단 기준 |
|------|------|-----------|
| 지민 | PD (프로젝트 디렉터) | 일정 준수, 리소스 현실성 |
| 현우 | 기획자 ① (사용자 리서치) | 사용자 문제 해결, 검증된 니즈 |
| 소연 | 기획자 ② (비즈니스 전략) | 비즈니스 임팩트, KPI |
| 태준 | 개발자 ① (백엔드) | 구현 가능성, 성능, 보안 |
| 미래 | 개발자 ② (프론트엔드) | 구현 속도, 사용자 체감 성능 |
| 윤서 | QA | 테스트 가능성, 릴리즈 안전성 |
| 다은 | UI 디자이너 | 브랜드 일관성, 컴포넌트 재사용 |
| 승호 | UX 디자이너 | 사용자 여정, 인지 부하, 접근성 |

### 팀원 프로필 구조

```markdown
### 이름 (직군)
- **역할**: 한 줄 설명
- **판단 기준**: 결정 시 중요하게 보는 것
- **분석 포커스**: 항목 목록
- **결과물 형식**: 어떤 형태로 의견을 낼 것인가
```

## 5. 파일 구조 (현재)

```
~/.claude/                              # 글로벌 (정규직)
├── commands/
│   ├── meeting-multi.md
│   ├── meeting-agent.md
│   └── meeting-team.md
└── meeting-team-profiles.md

{project}/.claude/                      # 프로젝트 (비정규직)
├── commands/
│   └── meeting-team.md
└── meeting-team-profiles.md
```

## 6. GUI 구현 목표

CLI로 검증한 개념을 **시각적 인터페이스**로 제공.

### 화면 구성 (초안)

```
┌─────────────────────────────────────┐
│  인사팀                               │
├────────────────┬────────────────────┤
│  정규직 (글로벌) │  비정규직 (프로젝트) │
│                │                    │
│  지민  (PD)    │  [이 프로젝트 전용]  │
│  현우  (기획)  │                    │
│  태준  (BE)    │  + 계약직 채용      │
│  ...           │                    │
│  + 정규직 채용  │                    │
├─────────────────────────────────────┤
│  오늘의 프리랜서                       │
│  [회의 시작 시 자동 고용 → 종료 후 소멸] │
├─────────────────────────────────────┤
│  예상 인건비: ~11,600 토큰 (4x)       │
│  회의 방식: [multi] [agent] [team]   │
└─────────────────────────────────────┘
```

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 팀원 카드 | 이름, 직군, 성격 요약 표시 |
| 고용 / 해고 | 클릭으로 팀 구성 |
| 발령 | 글로벌 ↔ 프로젝트 이동 |
| 예상 비용 | 선택 인원 × 방식 → 토큰 예측 |
| 회의 실행 | 방식 선택 + 주제 입력 → 실행 |
| 팀원 추가 | 폼으로 프로필 파일에 자동 기록 |
