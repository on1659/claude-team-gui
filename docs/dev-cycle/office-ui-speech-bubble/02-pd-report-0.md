3. Phase A(핵심 UI) 완료 후 Gate 검증 → Phase B(폴리싱) 진행

- 회의 중 상태별 시각 피드백 (idle=정지, streaming=talk, done=완료 이펙트)
- 스트리밍 중 말풍선에 마지막 문장/요약 표시

## 예상 사용 시나리오

### 개발 전 (As-Is)
1. 사용자가 VS Code에서 회의를 시작한다
2. 팀원 8명이 카드 형태로 나열된다 (이름 + 상태 뱃지)
3. 각 카드에 스트리밍 텍스트가 쏟아지지만, 누가 무슨 말을 하는지 시각적으로 구분이 어렵다
4. 회의가 끝나면 각 카드의 결과를 하나씩 펼쳐 읽어야 한다

### 개발 후 (To-Be)
1. 사용자가 VS Code에서 회의를 시작한다
2. 픽셀 오피스 뷰에 8명의 캐릭터가 자리에 앉아 있다
3. 발언 중인 캐릭터 위에 말풍선이 나타나고, 캐릭터가 talk 애니메이션을 한다
4. 말풍선에는 현재 발언의 핵심 문장이 1~2줄로 요약 표시된다
5. 우측 채팅 로그 패널에서 전체 대화 기록을 실시간으로 확인할 수 있다
6. 회의가 끝나면 채팅 로그를 스크롤하며 전체 맥락을 파악한다

### 기대 효과
- **시각적 직관성**: 카드 나열 → 오피스 공간 + 캐릭터 인격화
- **실시간 파악**: 누가 발언 중인지 즉시 식별 (카드 상태 뱃지 → 말풍선 + 애니메이션)
- **대화 기록 접근성**: 각 카드 펼치기 → 통합 채팅 로그 패널에서 한눈에 확인
