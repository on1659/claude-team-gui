# 팀원 — 고용 체계 및 구성

## 고용 형태

| 고용 형태 | 기술적 실체 | 범위 | 특징 |
|-----------|------------|------|------|
| **정규직** | `~/.claude/` 글로벌 팀원 | 전체 컴퓨터 | 모든 프로젝트 출근, 해고 전까지 유지 |
| **비정규직** | `.claude/` 프로젝트 팀원 | 프로젝트 한정 | 프로젝트 종료 시 계약 만료 |
| **프리랜서** | 서브에이전트 (일회성) | 세션 내 | 일 맡기면 하고 사라짐 |
| **재계약** | `resume` 파라미터 | 세션 내 | 프리랜서 재소환, 이전 컨텍스트 유지 |

부가 용어:
- **연봉** → 토큰 사용량
- **부서** → 역할 그룹 (기획팀 / 개발팀 / 디자인팀)
- **발령** → 글로벌 ↔ 프로젝트 이동
- **퇴사** → 프로필 삭제

## 현재 팀원 구성 (기본 8명)

| 이름 | 직군 | 판단 기준 | 분석 포커스 |
|------|------|-----------|-------------|
| 지민 | PD (프로젝트 디렉터) | 일정 준수, 리소스 현실성 | 일정 영향, 병목, 스코프 크립 |
| 현우 | 기획자 ① | 사용자 문제 해결, 검증된 니즈 | 사용자 스토리, 기능 범위, 성공 지표 |
| 소연 | 기획자 ② | 비즈니스 임팩트, KPI | 비즈니스 케이스, 경쟁사, 출시 전략 |
| 태준 | 개발자 ① (백엔드) | 구현 가능성, 성능, 보안 | API 설계, 성능 병목, 기술 부채 |
| 미래 | 개발자 ② (프론트엔드) | 구현 속도, 사용자 체감 성능 | 컴포넌트 설계, 렌더링 성능, 상태 관리 |
| 윤서 | QA | 테스트 가능성, 릴리즈 안전성 | 엣지케이스, 회귀 리스크, QA 공수 |
| 다은 | UI 디자이너 | 브랜드 일관성, 컴포넌트 재사용 | 디자인 시스템, 시각적 계층, 반응형 |
| 승호 | UX 디자이너 | 사용자 여정, 인지 부하, 접근성 | 태스크 흐름, 인터랙션, WCAG |

## 파일 구조

```
claude_team_gui/
├── .claude/
│   ├── commands/
│   │   ├── meeting-multi.md        # 역할극 8인
│   │   ├── meeting-agent.md        # 서브에이전트 병렬
│   │   └── meeting-team.md         # 팀원 프로필 기반
│   └── meeting-team-profiles.md    # 팀원 DB

~/.claude/                          # 글로벌 (정규직)
├── commands/
│   ├── meeting-multi.md
│   ├── meeting-agent.md
│   └── meeting-team.md
└── meeting-team-profiles.md
```

## 팀원 추가 방법

`.claude/meeting-team-profiles.md`에 아래 형식으로 추가:

```markdown
### 이름 (직군)
- **역할**: 한 줄 설명
- **판단 기준**: 이 역할이 결정할 때 중요하게 보는 것
- **분석 포커스**:
  - 항목1
  - 항목2
- **결과물 형식**: 어떤 형태로 의견을 낼 것인가
```
