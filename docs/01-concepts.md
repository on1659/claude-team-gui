# 개념 — Claude 팀원 시스템

## Claude 에이전트란?

Claude는 단일 인스턴스가 아니라 **여러 인스턴스가 협업**하는 구조로 운용할 수 있다.
오케스트레이터(주 Claude)가 Agent 도구로 서브에이전트를 생성하고, 작업을 위임한다.

```
오케스트레이터 Claude
    ↓ Agent 도구
서브에이전트 A  서브에이전트 B  서브에이전트 C
(독립 컨텍스트) (독립 컨텍스트) (독립 컨텍스트)
    ↓               ↓               ↓
        결과 반환 → 오케스트레이터 종합
```

## 서브에이전트 수명

에이전트는 **일회용(프리랜서)**이 기본이다.

```
생성 → 작업 실행 → 결과 반환 → 소멸
```

단, `resume` 파라미터로 재소환 가능:
```
1차 실행 → agent_id: "abc123" 반환
resume: "abc123" → 이전 컨텍스트 그대로 이어서 실행
```

## 역할극 vs 서브에이전트

### 역할극 방식 (meeting-multi)

```
[Claude 1개, 단일 컨텍스트]
→ 기획자 역할 발언  (주제만 앎)
→ 개발자 역할 발언  (기획자 말을 읽고 답변)
→ QA 역할 발언      (기획자+개발자 말을 읽고 답변)
```

- 앞 발언이 뒤 역할에 **영향을 줌 → 편향 발생 가능**
- 1인 다역 배우 구조
- 인원이 늘수록 컨텍스트 누적 → 토큰 폭증

### 서브에이전트 방식 (meeting-agent / meeting-team)

```
[오케스트레이터]
    ↓ 동시 실행
[기획자] [개발자] [QA] [UI] [UX] [PD] ...
 주제만    주제만   주제만  주제만
 독립분석  독립분석  독립분석  독립분석
```

- 서로의 발언을 **전혀 모르는 상태에서 독립 분석**
- 편향 없음
- 인원 증가 → 토큰 선형 증가

## 팀원 vs 서브에이전트

| | 서브에이전트 | 팀원 |
|---|---|---|
| 수명 | 작업 중에만 | 프로필로 지속 |
| 정체성 | 없음 (역할만) | 있음 (이름, 성격, 말투) |
| 기억 | 없음 | 프로필 파일로 유지 |
| 범위 | 세션 내 | 글로벌 / 프로젝트 |

팀원도 기술적으로는 서브에이전트다. 다만 프로필 파일에 정의된 정체성을 주입해서 **사람처럼 느껴지게** 설계된 서브에이전트다.
