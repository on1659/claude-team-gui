# `/meeting` — 경량 회의 (bkit 내장)

---

## 개요

bkit 내장 스킬. Claude 1개가 소수 역할을 순차 실행하는 가장 가벼운 회의 방식.

---

## 동작 방식

```
[Claude 1개]

→ 역할 A 발언
→ 역할 B 발언  ← A 발언 읽고 답변
→ 역할 C 발언  ← A+B 발언 읽고 답변
→ 요약
```

- 단일 컨텍스트 공유
- 순차 실행 (병렬 없음)
- 역할 수 적을수록 효율적

---

## 출력 구조 예시

> 주제: "모바일 앱에 다크모드 추가"

```
**기획자**: 사용자 요청이 많은 기능. OLED 배터리 절약 효과도 있어
타깃 사용자 중 야간 사용 비율 높으므로 우선순위 상.

**개발자**: React Native에서 useColorScheme으로 구현 가능.
기존 컴포넌트에 theme prop 추가 필요. 공수 약 3일.

**QA**: 라이트/다크 전환 시 색상 누락 케이스 체크 필요.
기기별 시스템 설정 연동도 테스트 항목.

---
결론: 구현 가능, 3일 공수. 시스템 설정 연동 포함해서 진행.
```

---

## 특징

| 항목 | 내용 |
|------|------|
| 토큰 | ~2,900 (8인 대비 1/4) |
| 속도 | 가장 빠름 |
| 편향 | 있음 (앞 발언 영향) |
| 인원 | 2~4명 권장 |
| 깊이 | 낮음 — 빠른 체크용 |

---

## 다른 방식과의 차이

- `/meeting-multi` 대비: 인원 적고 출력 짧음. 토론 흐름보다 빠른 요약.
- `/meeting-agent` 대비: 병렬 없음, 편향 있음, 훨씬 저렴.
- `/meeting-team` 대비: 팀원 정체성 없음, 역할만.

---

## 적합한 상황

- 빠르게 2~3가지 관점만 확인하고 싶을 때
- 토큰 아낄 때
- 결과보다 과정이 덜 중요할 때
