# `/meeting-team` — 팀원 프로필 기반 회의

---

## 개요

`profiles.md`에 정의된 팀원의 **이름, 성격, 말투, 판단 기준**을 주입한 서브에이전트를 병렬 실행하는 방식. `/meeting-agent`와 구조는 같지만, 역할(기능)이 아닌 **사람(정체성)**으로 동작한다.

---

## 동작 방식

```
[오케스트레이터]
  1. profiles.md 읽기
  2. 참여 팀원 선택

        ↓ Agent 도구로 동시 실행

[지민·PD]   [현우·기획①]  [소연·기획②]  [태준·BE]  ...
"당신은      "당신은        "당신은        "당신은
 지민입니다.  현우입니다.    소연입니다.    태준입니다.
 [프로필]"    [프로필]"      [프로필]"      [프로필]"

        ↓ 모든 결과 수집
[오케스트레이터] → 최종 종합
```

각 에이전트에게 프로필 전체가 주입됨 → 판단 기준, 말투, 분석 포커스가 일관됨.

---

## 출력 구조 예시

> 주제: "모바일 앱에 다크모드 추가"

각 팀원 개별 출력:
```
[지민 - PD]
일정 보면 이번 분기 가능은 한데, 테마 시스템 범위에 따라 달라져요.
컴포넌트 전수 수정이면 버퍼 2주 추가 필요. 범위부터 확정해야 Go 줄 수 있어요.
→ 우려: 스코프 크립 리스크 / 다음 액션: 범위 확정 회의 필요

[현우 - 기획자①]
사용자 리서치에서 야간 사용 시 불편함 호소 비율이 꽤 높아요.
특히 OLED 폰 쓰는 20~30대 타깃층. 검증된 니즈입니다.
→ 성공 지표: 야간 세션 이탈률 감소로 측정하면 어떨까요?

[태준 - 백엔드]
서버 쪽은 크게 영향 없어요. 설정값 저장 API 하나 추가하면 됨.
프론트가 테마 시스템을 어떻게 잡느냐가 핵심이고, 그쪽 공수가 변수예요.
보안상 이슈도 없고. 공수 산정: 백엔드 1일.
```

오케스트레이터 종합 출력:
```
## 🏁 다크모드 추가 — 팀 회의 결과

### 팀원별 의견
**지민 (PD)**: 이번 분기 가능하나 범위 확정 선행 필요. 스코프 크립 주의.
**현우 (기획①)**: 검증된 니즈. 야간 이탈률로 KPI 측정 제안.
**소연 (기획②)**: MVP로 시스템 연동만. 경쟁사 대비 빠른 출시 중요.
**태준 (백엔드)**: 서버 공수 1일. 프론트 테마 시스템이 변수.
**미래 (프론트)**: color token 시스템 없으면 공수 2배. 선행 설계 필요.
**윤서 (QA)**: 플리커, 시스템 연동, 기기 다양성 테스트 필수. QA 공수 3일.
**다은 (UI)**: 다크 팔레트 정의 + 디자인 시스템에 token 추가 작업 필요.
**승호 (UX)**: 설정 진입 UX, 즉시 적용 vs 재시작 여부 결정 필요.

### 주요 합의점
- 구현 가치 있음, 이번 분기 가능
- 색상 토큰 시스템 선행이 핵심

### 주요 충돌 지점
- **소연 vs 미래**: 빠른 출시(MVP) vs 테마 시스템 먼저
- **지민 vs 현우**: 일정 리스크 우선 vs 사용자 니즈 우선

### 서로에게 던진 질문들        ← meeting-agent에는 없는 항목
- 지민 → 미래: 테마 시스템 없이 MVP 가능한가?
- 현우 → 윤서: QA 기간 줄이는 방법 있는가?
- 다은 → 승호: 설정 화면 UX 같이 작업할 수 있는가?

### 권장 다음 액션
1. 범위 확정 회의 (지민, 소연, 미래) — 이번 주 내
2. 색상 토큰 설계 (다은 + 미래)
3. KPI 측정 방법 구체화 (현우)
```

## 특징

| 항목 | 내용 |
|------|------|
| 토큰 (8인) | ~18,100 (프로필 주입으로 input 증가) |
| 속도 | 병렬 실행 — 빠름 |
| 편향 | **없음** — 독립 컨텍스트 |
| 정체성 | **있음** — 이름, 성격, 말투, 판단 기준 일관 유지 |
| 출력 느낌 | 실제 팀원이 말하는 것처럼 느껴짐 |

---

## `/meeting-agent`와의 차이

| 항목 | agent | team |
|------|-------|------|
| 정체성 | 없음 (역할만) | 있음 (이름+성격+말투) |
| 일관성 | 실행마다 다를 수 있음 | 프로필 기반 일관 유지 |
| 추가 항목 | 없음 | "서로에게 던진 질문들" |
| 토큰 | ~11,600 | ~18,100 (프로필 토큰 추가) |
| 느낌 | 전문가 분석 보고서 | 실제 팀 회의록 |

---

## 적합한 상황

- 중요한 의사결정 (팀원 캐릭터가 신뢰감 줌)
- 팀원 구성이 고정되어 있고 일관성이 중요할 때
- 팀원 간 충돌 지점과 질문들을 추출하고 싶을 때
- 실제 팀 회의 시뮬레이션이 필요할 때
