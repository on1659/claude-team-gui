# `/meeting-agent` — 멀티에이전트 병렬 회의

---

## 개요

역할별로 **독립된 Claude 인스턴스**를 동시에 실행하는 방식. 각 에이전트는 주제만 알고 다른 역할의 발언을 전혀 모른 채 분석한다. 편향이 없고, 인원 증가 시 토큰이 선형으로 늘어난다.

---

## 동작 방식

```
[오케스트레이터 Claude]
        ↓ Agent 도구로 동시 실행

[기획자①] [기획자②] [개발자①] [개발자②] [QA] [UI] [UX] [PD]
    ↓          ↓          ↓          ↓       ↓    ↓    ↓    ↓
 주제만 알고  주제만 알고  주제만 알고  주제만 알고  ...
 독립 분석    독립 분석    독립 분석    독립 분석

        ↓ 모든 결과 수집
[오케스트레이터 Claude] → 최종 종합
```

각 에이전트는 서로 완전히 격리된 컨텍스트에서 실행된다.

---

## 출력 구조 예시

> 주제: "모바일 앱에 다크모드 추가"

각 에이전트 개별 출력 (병렬 실행):
```
[기획자① 분석]               [기획자② 분석]
1. 핵심 의견:                 1. 핵심 의견:
   야간 사용자 니즈 높음.        경쟁사 차별화 포인트.
   OLED 배터리 효과 있음.       이미 경쟁사 전원 지원 중.

2. 우려사항:                  2. 우려사항:
   기존 컴포넌트 수정 범위.      출시 지연 시 경쟁 열위.

3. 제안:                      3. 제안:
   이번 분기 출시 목표.         MVP로 시스템 다크/라이트만 지원.
```

오케스트레이터 종합 출력:
```
## 🏁 팀 회의 결과: 다크모드 추가

### 역할별 핵심 의견
**기획자①**: 야간 사용자 니즈 + OLED 효과. 이번 분기 우선순위 상.
**기획자②**: 경쟁사 전원 지원. MVP 범위(시스템 연동)로 빠른 출시 제안.
**개발자①**: React Native useColorScheme 활용. 테마 시스템 선행 필요.
**개발자②**: Async Storage 설정 저장. 공수 약 5일.
**QA**: 라이트↔다크 전환 플리커, 기기별 시스템 설정 연동 테스트 필수.
**UI**: 디자인 시스템에 color token 추가. 다크 팔레트 정의 필요.
**UX**: 설정 진입 경로, 즉시 반영 vs 재시작 UX 결정 필요.
**PD**: 3주 내 가능. 테마 시스템 범위가 일정 변수.

### 공통 합의점
- 구현 가치 있음
- 테마 시스템(color token) 선행이 핵심

### 주요 쟁점
- MVP 범위: 시스템 설정 연동만 vs 인앱 토글까지
- 테마 시스템 공수 포함 여부 (일정 영향)

### 권장 다음 액션
1. 색상 토큰 시스템 범위 확정 (개발자 + UI)
2. MVP 범위 결정 (기획자 + PD)
3. QA 테스트 기기 목록 확정
```

## 특징

| 항목 | 내용 |
|------|------|
| 토큰 (8인) | ~11,600 (multi 대비 40% 저렴) |
| 속도 | **병렬 실행 — 가장 빠름** |
| 편향 | **없음** — 완전 독립 컨텍스트 |
| 인원 증가 시 | 선형 증가 |
| 출력 느낌 | 각자 독립적이라 의견이 더 다양하고 날카로움 |

---

## `/meeting-multi`와의 핵심 차이

같은 주제, 같은 역할 구성이어도 결과가 다르다:

| 항목 | multi | agent |
|------|-------|-------|
| 개발자 발언 | 기획자 의견을 읽은 뒤 반응 | 기획자 의견 모름, 순수 기술 관점만 |
| QA 발언 | 앞의 모든 발언 읽고 정리 | 주제만 보고 독립적으로 리스크 발굴 |
| 의견 다양성 | 앞 발언 수렴 경향 | 더 다양하고 날카로운 의견 가능 |

---

## 적합한 상황

- 편향 없는 독립 의견이 필요할 때
- 인원 5명 이상 (토큰 효율)
- 중요한 아키텍처/방향 결정
- 결과를 빠르게 받고 싶을 때
