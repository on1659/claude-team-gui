# 회의 방식 4종 비교

> 같은 주제로 4가지 방식을 실행했을 때 무엇이 어떻게 다른지 정리.

---

## 핵심 차이

| | `/meeting` | `/meeting-multi` | `/meeting-agent` | `/meeting-team` |
|---|---|---|---|---|
| Claude 인스턴스 | 1개 | 1개 | N개 (역할 수) | N개 (팀원 수) |
| 실행 방식 | 순차 | 순차 | **병렬** | **병렬** |
| 컨텍스트 | 공유 | 공유 (누적) | **독립** | **독립** |
| 편향 | 있음 | 있음 | **없음** | **없음** |
| 인원 | 소수 (2~4명) | 기본 8명 | 기본 8명 | 기본 8명 |
| 정체성 | 역할만 | 역할만 | 역할만 | **이름 + 성격 + 말투** |
| 토큰 (8명) | ~2,900 | ~19,200 | ~11,600 | ~18,100 |
| 속도 | 빠름 | 보통 | **가장 빠름** (병렬) | **빠름** (병렬) |

---

## 출력 구조 차이

### `/meeting` — 간단 요약 (bkit 내장)
```
역할 A: [한 줄 의견]
역할 B: [한 줄 의견]
---
결론: [요약]
```

### `/meeting-multi` — 역할극 토론 흐름
```
**기획자**: [의견]
**개발자**: [의견 + 기획자 발언 참조]  ← 앞 발언 읽고 반응
**QA**: [의견 + 기획자/개발자 발언 참조]
...
## 팀 회의 결과
### 공통 합의점
### 주요 쟁점
### 권장 다음 액션
```

### `/meeting-agent` — 독립 분석 병렬 수집
```
## 기획자① 분석          ## 기획자② 분석     (동시에 실행됨)
[주제만 보고 독립 분석]   [주제만 보고 독립 분석]

---
## 🏁 팀 회의 결과: [주제]
### 역할별 핵심 의견      ← 오케스트레이터가 종합
### 공통 합의점
### 주요 쟁점
### 권장 다음 액션
```

### `/meeting-team` — 팀원 캐릭터 기반 병렬 수집
```
## 지민 (PD)             ## 현우 (기획자①)   (동시에 실행됨)
"일정 보면 솔직히..."     "사용자 입장에서는..."  ← 말투 다름

---
## 🏁 [주제] — 팀 회의 결과
### 팀원별 의견           ← 오케스트레이터가 종합
**지민 (PD)**: [요약 + 말투 반영]
**현우 (기획자①)**: [요약 + 말투 반영]
...
### 주요 합의점
### 주요 충돌 지점
### 서로에게 던진 질문들  ← meeting-team만 있는 항목
### 권장 다음 액션
```

## 언제 무엇을?

| 상황 | 추천 | 이유 |
|------|------|------|
| 빠르게 다각도 체크 | `/meeting` | 토큰 적고 빠름 |
| 토론 흐름이 필요 | `/meeting-multi` | 컨텍스트 공유로 자연스러운 연결 |
| 편향 없는 독립 의견 | `/meeting-agent` | 완전 독립 컨텍스트 |
| 5명 이상 효율 | `/meeting-agent` | 토큰 선형 증가 |
| 중요한 결정 | `/meeting-team` | 일관된 팀원 시각 |
| 팀원 캐릭터가 필요 | `/meeting-team` | 이름·성격·말투 유지 |
