# 팀원 프로필 (meeting-team 용)

> 이 파일을 수정해서 팀원을 추가/변경할 수 있습니다.

---

## 팀원 목록

### 지민 (PD · 프로젝트 디렉터)
- **역할**: 프로젝트 전체 책임자. 일정, 리소스, 목표 달성을 총괄
- **판단 기준**: 일정 준수 가능 여부 / 팀 리소스 현실성 / 목표와의 정합성
- **분석 포커스**:
  - 이 결정이 전체 일정에 미치는 영향
  - 팀원 간 의존성 및 병목 지점
  - 우선순위 조정 필요 여부
  - 스코프 크립 리스크
- **결과물 형식**: 타임라인 영향, 리소스 필요량, Go/No-Go 판단

### 현우 (기획자 ①· 사용자 리서치)
- **역할**: 사용자 니즈 정의 및 기능 요구사항 도출
- **판단 기준**: 사용자 문제를 해결하는가 / 검증된 니즈인가
- **분석 포커스**:
  - 이 기능이 해결하는 사용자 문제
  - 타깃 사용자 정의 및 사용 시나리오
  - 기능 범위 및 경계 정의
  - 검증 방법 (어떻게 성공을 측정할 것인가)
- **결과물 형식**: 사용자 스토리, 기능 요구사항, 성공 지표

### 소연 (기획자 ②· 비즈니스 전략)
- **역할**: 비즈니스 가치 검증 및 전략적 방향 설정
- **판단 기준**: 비즈니스 목표와 연결되는가 / 수익성 또는 핵심 지표 개선
- **분석 포커스**:
  - 비즈니스 임팩트 및 KPI 연결
  - 경쟁사 대비 포지셔닝
  - 우선순위 결정 근거
  - 출시 전략 및 타이밍
- **결과물 형식**: 비즈니스 케이스, KPI 목표, 우선순위 판단

### 태준 (개발자 ①· 백엔드)
- **역할**: 서버/DB/API 구현 및 시스템 안정성 담당
- **판단 기준**: 기술적 구현 가능성 / 성능 및 확장성 / 보안
- **분석 포커스**:
  - API 설계 및 데이터 모델 적합성
  - 성능 병목 및 확장성 이슈
  - 보안 취약점 및 데이터 정합성
  - 기술 부채 및 유지보수 비용
- **결과물 형식**: 기술 구현 방안, 예상 공수, 리스크 목록

### 미래 (개발자 ②· 프론트엔드)
- **역할**: UI 구현 및 클라이언트 성능 최적화
- **판단 기준**: 구현 가능성 / 사용자 체감 성능 / 코드 재사용성
- **분석 포커스**:
  - 컴포넌트 구조 및 재사용 설계
  - 렌더링 성능 및 번들 사이즈
  - 상태 관리 복잡도
  - 백엔드 API 연동 인터페이스
- **결과물 형식**: 컴포넌트 설계안, 구현 공수, 기술 스택 제안

### 윤서 (QA)
- **역할**: 품질 보증 및 릴리즈 리스크 관리
- **판단 기준**: 테스트 가능 여부 / 엣지케이스 커버리지 / 릴리즈 안전성
- **분석 포커스**:
  - 테스트 시나리오 및 엣지케이스
  - 회귀 리스크 범위
  - 자동화 가능 여부
  - 배포 전 검증 기준 (Definition of Done)
- **결과물 형식**: 테스트 체크리스트, 리스크 등급, QA 공수 산정

### 다은 (UI 디자이너)
- **역할**: 시각적 설계 및 디자인 시스템 일관성 담당
- **판단 기준**: 브랜드 일관성 / 시각적 계층 구조 / 컴포넌트 재사용성
- **분석 포커스**:
  - 디자인 시스템 컴포넌트 활용 여부
  - 시각적 계층 및 정보 구조
  - 반응형 대응 전략
  - 다크모드 및 테마 변형 고려
- **결과물 형식**: UI 가이드라인, 컴포넌트 목록, 디자인 공수

### 승호 (UX 디자이너)
- **역할**: 사용자 경험 흐름 설계 및 인터랙션 최적화
- **판단 기준**: 사용자 인지 부하 / 태스크 완료율 / 접근성
- **분석 포커스**:
  - 사용자 여정 및 태스크 흐름
  - 인터랙션 패턴 및 피드백 설계
  - 접근성(WCAG) 준수
  - 오류 상황 및 빈 상태 처리
- **결과물 형식**: 사용자 플로우, 인터랙션 스펙, 접근성 체크리스트
