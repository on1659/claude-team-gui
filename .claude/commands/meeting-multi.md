# /meeting-multi - 역할극 기반 팀 회의 (8인)

## 개요
Claude 1개가 여러 역할을 순차적으로 연기하는 역할극 방식.
컨텍스트를 공유하므로 앞 발언이 뒤 역할에 영향을 줌.

기본 구성: 기획자x2, 개발자x2, QA x1, UI x1, UX x1, PD x1 (총 8명)

---

## Step 1: 회의 설정 확인

AskUserQuestion 도구로 다음을 확인하세요:

**질문 1 - 회의 주제:**
- $ARGUMENTS 가 있으면 그걸 주제로 사용
- 비어있으면 주제 입력 요청

**질문 2 - 역할 구성:**
기본값: `기획자x2, 개발자x2, QA x1, UI x1, UX x1, PD x1` (총 8명)
변경하고 싶으면 자유 입력.

---

## Step 2: 역할극 순차 실행

각 역할을 **순서대로** 연기하세요. 매 역할마다 이전 발언을 읽고 답변.

각 역할 발언 형식:
```
---
**[역할명]**
[핵심 의견 — 3줄 이내]
[우려사항 또는 리스크]
[제안 방향]
---
```

역할별 포커스:
- **기획자**: 사용자 니즈, 기능 범위, 요구사항
- **개발자**: 구현 가능성, 기술 스택, 공수
- **QA**: 품질 리스크, 테스트 범위, 엣지케이스
- **UI**: 시각 설계, 컴포넌트, 디자인 시스템
- **UX**: 사용자 흐름, 인터랙션, 접근성
- **PD**: 일정, 리소스, 우선순위

---

```
## 🏁 팀 회의 결과: [주제]

### 공통 합의점
### 주요 쟁점
### 권장 다음 액션
```
