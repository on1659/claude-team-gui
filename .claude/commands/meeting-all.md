# /meeting-all - 3종 회의 일괄 실행 + 비교 분석

## 개요
meeting-multi, meeting-agent, meeting-team을 동일 주제로 순차 실행한 뒤,
04-meeting-result (비교 분석 + 점수표 + 구현 추천) 문서까지 자동 생성.

**토큰 사용량을 실측하여 각 회의/에이전트별 비용을 기록.**

---

## Step 1: 회의 설정

AskUserQuestion 도구로 다음을 확인하세요:

**질문 1 - 회의 주제:**
- $ARGUMENTS 가 있으면 그걸 주제로 사용
- 비어있으면 주제 입력 요청

**질문 2 - 저장 폴더:**
- 기본값: `docs/meetings/[주제-slug]/`
- 사용자가 변경 가능

**질문 3 - 참여 팀원 (meeting-team용):**
- 기본값: 전원 (지민, 현우, 소연, 태준, 미래, 윤서, 다은, 승호)
- 변경 가능

## Step 2: meeting-multi 실행

### 2-1. 역할극 순차 실행

`/meeting-multi`와 동일하게 8역할을 순차 연기.
각 역할: 핵심 의견 + 우려사항 + 제안 방향.

### 2-2. 결과 저장

`[폴더]/01-meeting-multi.md`로 저장.

문서 헤더에 토큰 기록:
```
> 방식: 역할극 (단일 API)
> 토큰: 입력 ~[X]K / 출력 ~[Y]K / 합계 ~[Z]K
> 비용 추정: ~$[금액]
```

## 주의사항

1. **순서 필수**: multi → agent → team → result. multi가 가장 가볍고 agent/team은 무거움.
2. **저장 먼저**: 각 회의 결과를 파일로 먼저 저장한 후 다음 회의 진행. 컨텍스트 관리를 위해.
3. **토큰 실측**: 각 단계에서 실제 사용된 토큰을 기록. 추정이 아닌 실측 데이터 우선.
4. **컨텍스트 한계**: 3종 모두 실행하면 토큰 소모가 크므로, 각 회의 결과를 파일로 저장 후 이전 회의의 상세 내용은 요약으로 교체하여 컨텍스트 관리.
5. **result 문서의 프롬프트 기록**: 04-meeting-result.md 첫 부분에 원본 프롬프트 그대로 기록.
