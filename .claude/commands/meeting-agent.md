# /meeting-agent - 실제 멀티에이전트 팀 회의

## 개요
실제 Agent 도구를 사용해 역할별 독립 에이전트를 병렬 실행하는 팀 회의.
`/meeting-multi`와 달리 각 에이전트가 완전히 독립된 컨텍스트에서 실행됨.

---

## Step 1: 회의 설정 확인

AskUserQuestion 도구로 다음 2가지를 물어보세요:

**질문 1 - 회의 주제:**
- $ARGUMENTS 가 있으면 그걸 주제로 사용
- 비어있으면 주제 입력 요청

**질문 2 - 역할 구성:**
기본값: `기획자x2, 개발자x2, QA x1, UI x1, UX x1, PD x1` (총 8명)
사용자가 변경하고 싶으면 자유 입력 가능.
형식 예시: `기획자x1, 백엔드x2, 프론트x1, QA x1`

---

## Step 2: 에이전트 병렬 실행

확인된 역할 구성으로 **Agent 도구를 동시에(한 번에) 실행**하세요.

각 에이전트 프롬프트 구조:
```
당신은 [역할명] #[번호] 입니다.
회의 주제: [주제]

[역할]의 전문적 관점에서 다음을 작성하세요:
1. 핵심 의견 (3줄 이내)
2. 우려사항 또는 리스크
3. 제안하는 방향

다른 역할의 관점은 고려하지 말고, 본인 역할 관점만 집중해서 작성하세요.
```

역할별 포커스 가이드:
- **기획자**: 비즈니스 가치, 사용자 니즈, 일정/범위
- **개발자**: 기술 구현 가능성, 아키텍처, 복잡도
- **QA**: 품질 리스크, 테스트 범위, 엣지케이스
- **UI**: 시각적 표현, 컴포넌트 설계, 디자인 시스템
- **UX**: 사용자 흐름, 인터랙션, 접근성
- **PD**: 프로젝트 전체 조율, 일정/리소스 관리, 목표 달성 책임

---

## Step 3: 결과 종합

모든 에이전트 결과를 받은 후:

```
## 🏁 팀 회의 결과: [주제]

### 역할별 핵심 의견
[각 에이전트 요약]

### 공통 합의점
[여러 역할이 동의하는 부분]

### 주요 쟁점
[역할 간 의견 차이가 있는 부분]

### 권장 다음 액션
[구체적 실행 항목 3가지 이내]
```
