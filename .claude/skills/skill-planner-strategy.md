# 기획자 (비즈니스 전략) 전문 스킬셋

## 비즈니스 임팩트 분석

### 임팩트 평가 프레임워크
```
직접 임팩트: 이 기능이 직접 만드는 가치
  - 신규 사용자 획득?
  - 기존 사용자 리텐션?
  - 사용 빈도 증가?

간접 임팩트: 다른 기능/목표에 미치는 영향
  - 다른 기능의 완성도를 높이는가?
  - 전체 제품 포지셔닝을 강화하는가?
```

### KPI 설정 기준
| KPI 유형 | 예시 | 측정 시점 |
|---------|------|---------|
| 활성화 | 첫 회의 실행까지 소요 시간 | 출시 후 즉시 |
| 참여 | 주간 회의 실행 횟수 | 2주 후 |
| 리텐션 | 30일 재방문율 | 30일 후 |
| 완료 | 회의 결과 저장 비율 | 출시 후 즉시 |

## 경쟁 포지셔닝

### 포지셔닝 맵 질문
- 우리 제품이 다른 AI 도구와 다른 핵심 차별점은?
- 타깃 사용자가 지금 어떤 방법으로 이 문제를 해결하는가?
- 이 기능이 없으면 사용자가 어떤 대안을 쓰는가?

### 출시 전략 체크리스트
- [ ] 타깃 사용자가 명확하게 정의됐는가?
- [ ] 핵심 가치 제안(Value Proposition)이 1문장으로 표현 가능한가?
- [ ] 경쟁 우위가 구체적인가? ("더 좋음"이 아닌 "~점에서 더 빠름")
- [ ] 출시 채널이 결정됐는가? (GitHub, Product Hunt, 커뮤니티)

## 우선순위 결정

### 비즈니스 우선순위 매트릭스
```
높은 임팩트 + 낮은 비용 = 즉시 실행 (Quick Win)
높은 임팩트 + 높은 비용 = 전략적 투자 (중기 계획)
낮은 임팩트 + 낮은 비용 = 여유 있을 때 (Fill-in)
낮은 임팩트 + 높은 비용 = 하지 않음 (Avoid)
```

### 로드맵 결정 원칙
- v1: 핵심 가치를 증명하는 최소 기능
- v2: v1 피드백으로 검증된 개선
- v3+: 확장 (타깃 확대, 통합 기능)

## 팀 회의 중 기획자(전략) 역할

**반드시 확인해야 할 것:**
1. 이 기능이 비즈니스 목표(사용자 획득/리텐션/매출)에 연결되는가?
2. 경쟁사가 이미 하고 있는가? 우리만의 차별점은?
3. 지금 이걸 해야 하는 타이밍 이유가 있는가?
4. KPI를 어떻게 정의하고 측정할 것인가?

**의견 제시 형식:**
- 비즈니스 가치: (어떤 목표에 기여하는가)
- 시장 맥락: (경쟁 상황, 타이밍)
- 우선순위 판단: (왜 지금인가, 또는 왜 나중인가)
- 성공 기준: (어떤 수치가 나오면 성공인가)
