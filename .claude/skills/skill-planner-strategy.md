# 기획자② (소연) — 이 프로젝트 컨텍스트

## 확정된 결정
- **포지셔닝**: Claude Code 팀원 시스템의 GUI 인터페이스 — CLI 없이 클릭으로
- **타깃**: 개발자 먼저, 비개발자는 검증 후
- **배포**: Electron 패키징 (win/mac), 설치형

## 비즈니스 맥락
```
지금 상황: Claude Code의 멀티에이전트 회의 기능을 CLI로만 쓸 수 있음
차별점: GUI로 팀원을 시각적으로 관리하고 회의를 원클릭 실행
시장: AI 협업 툴이 폭발적으로 증가 중 → 빠른 출시가 중요
```

## 이 프로젝트 KPI
| 지표 | 목표 | 측정 시점 |
|------|------|---------|
| 첫 회의 실행까지 시간 | < 1분 | 출시 직후 |
| 주간 회의 실행 횟수 | > 3회/사용자 | 2주 후 |
| GitHub star | 출시 후 성장 추이 | 1달 후 |
| 회의 완료율 | > 80% | 출시 직후 |

## 로드맵
```
v1 (지금): 팀원 관리 표시 + 회의 실행 4종 + 토큰 예측
v2 (검증 후): 프로필 편집 UI, 팀원 추가/삭제, 회의 기록
v3 (확장): 비개발자 타깃, 웹앱 버전, 팀 단위 관리
```

## 출시 전략
- 채널: GitHub 공개 + Claude Code 사용자 커뮤니티
- 핵심 메시지: "Claude 팀원 시스템을 GUI로"
- v1 출시 기준: meeting-team 1종이 오류 없이 end-to-end 동작

## 경쟁 분석 프레임워크

```
비교 대상 (AI 협업 도구):
  직접 경쟁:
    - Claude Code CLI (/meeting-team 그 자체) — 우리의 베이스
    - Cursor Composer — AI 멀티파일 편집, 회의 기능 없음
    - Aider — CLI 기반 AI 페어 프로그래밍, 멀티에이전트 없음

  간접 경쟁:
    - ChatGPT Teams — 웹 기반 팀 대화, 역할 분담 없음
    - GitHub Copilot Workspace — 이슈→코드 자동화, 회의 없음

비교 축:
  | 축                | 우리 (v1)        | CLI 원본       | Cursor       |
  |-------------------|-----------------|----------------|--------------|
  | 멀티에이전트 회의   | ✓ GUI           | ✓ CLI          | ✗            |
  | 역할별 전문성      | ✓ 8명 프로필+스킬 | ✓ 프로필만      | ✗            |
  | 실시간 스트리밍     | ✓ 카드별         | ✓ 터미널        | ✓            |
  | 설치 난이도        | 중 (Electron)    | 상 (CLI 설정)   | 하 (IDE 통합) |
  | 비개발자 접근      | ✗ (v2)          | ✗              | ✗            |

차별화 포인트:
  1. 팀원을 "시각적으로" 관리 (카드 UI, 상태 표시)
  2. 원클릭 실행 (CLI 명령어 입력 불필요)
  3. 병렬 결과를 한눈에 비교 (그리드 레이아웃)
  4. 실패 에이전트만 선택 재시도 (CLI에서는 전체 재실행)
```
