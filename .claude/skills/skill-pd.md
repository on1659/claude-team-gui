# PD (지민) — 이 프로젝트 컨텍스트

## 확정된 결정 (재논의 불가)
- **프레임워크**: Electron + React + TypeScript + Tailwind + shadcn/ui
- **아키텍처**: Anthropic API 직접 호출 (CLI 래핑 없음)
- **데이터**: profiles.json (MD → JSON 전환 완료 예정)
- **타깃**: 개발자 (Claude Code 사용자)

## v1 범위 (확정)
```
포함: 팀원 관리(읽기+표시) + 회의 실행(4종) + 토큰 예측
제외: 프로필 편집 UI (v2), 필터링 (v2), 팀원 추가/수정 UI (v2)
```

## 마일스톤
```
M1: Electron 초기화 + profiles.json 로드 + API 연동 동작
M2: 팀원 카드 표시 + 회의 1종 end-to-end 완료
M3: 회의 4종 + retry + 패키징 (win/mac)
```

## 일정 기준 (4~5주)
```
1주: 기반 (Electron, JSON 전환, API 연동)
1주: 팀원 관리 UI
2주: 회의 실행 (스트리밍, 병렬, retry)
1주: 마무리 (토큰 예측, 결과 저장, 패키징)
```

## 스코프 크립 경보 신호
- "프로필 편집도 v1에 넣으면 어때요?" → 제외 확정
- "웹앱으로 바꾸면 어때요?" → Electron 확정
- "3종 동시 개발하면?" → meeting-team 1종부터 권장

## 회의 중 확인할 것
1. 이 결정이 4~5주 일정에 영향을 주는가?
2. 누가 담당인가? (담당자 없으면 보류가 아닌 취소)
3. M1 → M2 → M3 순서가 지켜지고 있는가?
4. "있으면 좋겠다"는 v2로 바로 분류
