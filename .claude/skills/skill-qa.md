# QA 전문 스킬셋

## 테스트 전략

### 테스트 피라미드
```
         /\
        /E2E\        ← 적게 (핵심 플로우만)
       /------\
      /통합 테스트\    ← 중간 (API, DB, 컴포넌트 연동)
     /----------\
    / 단위 테스트  \   ← 많이 (함수, 유틸리티)
   /--------------\
```

### 테스트 우선순위 결정
1. **핵심 경로**: 사용자가 가장 많이 쓰는 플로우
2. **높은 리스크**: 실패 시 데이터 손실, 보안 이슈
3. **자주 바뀌는 코드**: 회귀 위험 높음
4. **복잡한 로직**: 엣지케이스 많은 부분

## 엣지케이스 발굴 프레임워크

### CRUD 엣지케이스 매트릭스
```
Create: 빈 입력 / 최대 길이 초과 / 중복 / 특수문자
Read:   없는 ID / 권한 없음 / 빈 목록 / 대용량
Update: 동시 수정 / 일부 필드만 / 없는 항목에 수정
Delete: 없는 항목 / 관계된 데이터 / 소프트 삭제
```

### API 통신 엣지케이스
- 네트워크 끊김 중 요청
- 응답 지연 (30초 이상)
- 부분 응답 (스트리밍 중단)
- 레이트 리밋 초과 (429)
- 서버 에러 (500)
- 잘못된 JSON 응답

### Anthropic API 특수 케이스 (이 프로젝트)
- 8 에이전트 중 3개만 성공한 경우
- 스트리밍 중 연결 끊김
- API 키가 만료된 경우
- 토큰 한도 초과 (max_tokens 도달)
- 모델 과부하 (과부하 에러)

## 테스트 체크리스트

### 기능 테스트
- [ ] Happy Path: 정상 동작 확인
- [ ] Error Path: 오류 발생 시 메시지 표시
- [ ] Edge Cases: 경계값, 빈값, 최대값
- [ ] Recovery: 오류 후 재시도 가능한가

### 비기능 테스트
- [ ] 성능: 8 에이전트 병렬 실행이 60초 내 완료
- [ ] 안정성: 10회 연속 실행 중 오류율 < 5%
- [ ] 호환성: Windows, macOS 양쪽 동작
- [ ] 보안: API 키가 로그/UI에 노출 안 됨

## Definition of Done (완료 기준)

### 기능 완료 기준
- [ ] 기능 요구사항 100% 구현
- [ ] 단위 테스트 작성 및 통과
- [ ] 핵심 E2E 시나리오 검증
- [ ] 코드 리뷰 완료
- [ ] 문서 업데이트

### 릴리즈 기준
- [ ] 모든 크리티컬/하이 버그 해결
- [ ] 성능 목표 달성
- [ ] 보안 체크리스트 통과
- [ ] 다음 버전과 데이터 호환성 확인

## 팀 회의 중 QA 역할

**반드시 확인해야 할 것:**
1. 이 기능의 실패 시나리오가 모두 정의됐는가?
2. 테스트 가능한 형태로 만들어졌는가? (인터페이스 명확, 모킹 가능)
3. 회귀 영향 범위가 어디인가? (다른 기능에 영향)
4. 이 기능의 DoD(완료 기준)가 명확한가?

**의견 제시 형식:**
- 테스트 시나리오: (Happy Path + 주요 엣지케이스)
- 리스크 등급: (Critical / High / Medium / Low)
- 회귀 범위: (영향받는 기존 기능)
- QA 공수: (몇 일, 어떤 도구로)
