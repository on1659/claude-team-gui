# UX 디자이너 (승호) — 이 프로젝트 컨텍스트

## 이 앱의 핵심 유저 플로우
```
앱 실행
  → (첫 실행) API 키 설정 → 완료
  → 팀원 목록 확인 (profiles.json에서 로드)
  → 팀원 선택
  → 주제 입력
  → 회의 방식 선택 (기본값: meeting-team)
  → [회의 시작] 클릭
  → 실행 중: 카드별 스트리밍 + "N/8 완료" 진행 표시
  → 전체 완료: 요약 패널 표시
  → 결과 저장 또는 새 회의
```

## 병렬 에이전트 실행 UX (확정)
```
방식: 카드별 독립 스트리밍 + 상단 전체 진행률 바
  → 상단: "6/8 완료" 프로그레스 바
  → 각 카드: 완료되는 순서대로 결과 표시 (순서 고정 필요)
  → 전체 완료 후: 요약 패널 펼쳐짐
```

## 회의 방식 선택 UX
```
회의 시작 시 1회 선택 — 런타임 전환 없음
4가지 방식의 차이를 5초 안에 파악할 수 있어야 함:
  meeting       — "빠른 방향 체크 (저렴)"
  meeting-multi — "토론 흐름 기록"
  meeting-team  — "심층 분석 + 다음 아젠다" ← 기본값
  meeting-agent — "편향 없는 의견 분포"
```

## 오류 UX 원칙 (이 앱)
```
API 키 없음:
  → 실행 버튼 비활성화 + "설정에서 API 키를 입력하세요" 안내

에이전트 실패 (카드):
  → "응답을 받지 못했어요. [재시도]"
  → retry 버튼은 해당 카드에만 표시

전체 타임아웃:
  → 토스트 "N개 에이전트가 응답하지 않았어요"
  → 받은 결과는 그대로 표시

좋은 에러 메시지: "API 키가 올바르지 않습니다. 설정에서 확인해주세요."
나쁜 에러 메시지: "오류가 발생했습니다 (401)"
```

## 피드백 타이밍 (이 앱 기준)
| 행동 | 피드백 | 언제 |
|------|--------|------|
| 팀원 선택 | 카드 하이라이트 | 즉시 |
| 회의 시작 클릭 | 버튼 로딩 + 카드 running 상태 | 즉시 |
| 에이전트 응답 | 카드에 텍스트 스트리밍 | 스트림 시작 시 |
| 에이전트 완료 | 완료 아이콘 + 진행률 +1 | 즉시 |
| 전체 완료 | 요약 패널 펼쳐짐 | 마지막 완료 즉시 |

## 첫 사용자 경험
```
1. 앱 실행 → API 키 없으면 설정 화면 먼저
2. 설정 완료 → 팀원 목록 (profiles.json 기반)
3. 팀원이 없으면 → "파일을 수정해 팀원을 추가하세요" 안내 (v1)
   (v2에서는 GUI로 추가)
```

## 회의 중 확인할 것
1. 사용자가 "지금 몇 명이 완료됐는가"를 항상 알 수 있는가?
2. 에이전트 실패 시 무엇을 해야 하는지 명확한가?
3. 회의 방식 4종의 차이가 선택 화면에서 바로 이해되는가?
4. 실수로 실행했을 때 취소가 가능한가?

## 의견 형식
- **플로우**: (사용자가 이 단계에서 무엇을 보고 무엇을 하는가)
- **피드백**: (어떤 시각적 반응이 필요한가)
- **오류 UX**: (실패 메시지 + 다음 행동 안내)
- **첫 사용자**: (온보딩 고려 필요 여부)
