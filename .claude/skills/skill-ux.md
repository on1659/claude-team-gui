# UX 디자이너 전문 스킬셋

## 사용자 여정 설계

### 태스크 플로우 분석 템플릿
```
[진입점] → [결정 포인트] → [행동] → [피드백] → [완료]

예: 회의 실행 플로우
  홈 화면 도착
    → 팀원 선택 (몇 명 선택?)
    → 주제 입력
    → 회의 방식 선택
    → 실행 버튼 클릭
    → 진행 중 피드백 (N/8 완료)
    → 결과 화면
    → 저장 또는 공유
```

### 인지 부하 줄이기
- **청킹**: 관련 정보를 그룹으로 묶기 (7±2 원칙)
- **점진적 공개**: 처음엔 핵심만, 나머지는 클릭 후 공개
- **기본값**: 가장 흔한 선택을 미리 선택 상태로
- **맥락 보존**: 페이지 이동 후에도 이전 작업 기억

## 인터랙션 설계

### 피드백 원칙 (사용자 행동 → 시스템 반응)
| 행동 | 피드백 | 지연 기준 |
|------|--------|---------|
| 버튼 클릭 | 즉각 반응 (색 변화) | < 100ms |
| 데이터 저장 | 성공 토스트 | < 200ms |
| API 호출 | 로딩 인디케이터 | > 300ms |
| 에러 발생 | 명확한 에러 메시지 | 즉시 |

### 오류 처리 UX
```
오류 메시지 작성 원칙:
1. 무엇이 잘못됐는가? (명확하게)
2. 왜 잘못됐는가? (가능하면)
3. 어떻게 해결하는가? (행동 가능한 안내)

좋은 예: "API 키가 올바르지 않습니다. 설정에서 키를 확인해주세요."
나쁜 예: "오류가 발생했습니다. (Error code: 401)"
```

### 빈 상태 처리
- 첫 사용자: 온보딩 안내 (무엇을 할 수 있는지)
- 검색 결과 없음: 검색어 제안 또는 필터 초기화
- 에러 후 빈 상태: 재시도 버튼 + 원인 안내

## 접근성 (WCAG 2.1 AA 기준)

### 키보드 접근성
- [ ] 모든 인터랙티브 요소가 Tab으로 접근 가능한가?
- [ ] 포커스 순서가 논리적인가? (상→하, 좌→우)
- [ ] 포커스 표시가 시각적으로 명확한가? (2px 이상 아웃라인)
- [ ] 모달: Escape 키로 닫힘, 트랩 포커스 적용

### 색상 접근성
- 텍스트 명도 대비: 최소 4.5:1 (일반), 3:1 (큰 텍스트)
- 색만으로 의미 전달 금지 (아이콘 또는 텍스트 병행)
- 색맹 사용자 고려 (red/green 동시 사용 주의)

### 스크린 리더
- 이미지: `alt` 속성 필수
- 아이콘 버튼: `aria-label` 필수
- 상태 변화: `aria-live` 영역으로 알림

## 회의 실행 UX 특화

### 병렬 에이전트 실행 UX
```
진행 상태 표시 방법:
  Option A: 카드별 로딩 인디케이터 (실시간, 복잡)
  Option B: 전체 프로그레스 바 N/8 (단순, 명확)
  Option C: 완료된 카드부터 순차 표시 (자연스러움)

권장: Option B + C 조합
  → 상단에 전체 진행률 표시
  → 완료된 팀원 카드는 결과 미리 보임
```

### 결과 표시 UX
- 요약 → 상세 순서로 (글로벌 → 로컬)
- 충돌 지점은 시각적으로 강조 (다른 색, 배지)
- 질문 목록은 다음 회의 아젠다로 즉시 사용 가능하게

## 팀 회의 중 UX 역할

**반드시 확인해야 할 것:**
1. 사용자가 현재 상태를 항상 알 수 있는가? (피드백)
2. 실수를 쉽게 되돌릴 수 있는가? (취소/되돌리기)
3. 처음 사용자도 도움 없이 핵심 기능을 완료할 수 있는가?
4. 오류 상황에서 사용자가 다음 행동을 알 수 있는가?

**의견 제시 형식:**
- 사용자 플로우: (단계별 행동과 피드백)
- 인터랙션 스펙: (구체적인 동작 방식)
- 접근성 포인트: (키보드, 색상 대비)
- 오류/빈 상태: (어떻게 처리할 것인가)
